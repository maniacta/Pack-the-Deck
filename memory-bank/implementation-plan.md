# Pack the Deck - 基础游戏实施计划

> **目标**：构建一个可玩的核心游戏循环（出牌→得分→过关）
> **范围**：牌型判断 + 战斗循环 + 基础装备 + 简单背包 + 商店系统
> **原则**：每步可测试、可验证、小步迭代

---

## 设计决策记录

以下设计决策已在开发前澄清：

| 决策项 | 结论 |
|--------|------|
| 牌型选择模式 | **玩家手动选牌**（最多选 5 张，系统识别这 5 张的牌型） |
| 盲注倍率含义 | 目标分数门槛提高，奖励提高 |
| Boss 特殊规则 | 某些花色不计分、某些牌型不计分、出牌数量限制等 |
| 装备形状旋转 | MVP 阶段不需要，后续迭代增加 |
| 规则改写实现 | **规则栈叠加** — 每个装备是一个规则层，按优先级依次应用 |
| 装备冲突规则 | **同类装备不能同时装备**（MVP 阶段） |
| 牌组规模 | 标准 52 张，装备效果可能改变牌组，动态管理 |
| 商店系统 | MVP 包含商店系统 |
| 卡牌视觉 | MVP 使用占位符 + 花色符号（♠♥♦♣） |

## 阶段三：得分计算系统

### 步骤 3.1：实现基础分数计算

**目标**：计算选出牌型的得分

**指令**：
1. 创建得分计算函数
2. 计算公式：
   ```
   得分 = 牌型基础分 × 牌型倍率
   牌型基础分 = 组成牌型的卡牌面值之和
   ```
3. 实现卡牌面值求和：
   - 数字牌 = 面值
   - J/Q/K = 10
   - A = 11（可被装备改写）

**验证测试**：
- [ ] 对子 [黑桃5, 红心5]：基础分 = 10，倍率 = 2，得分 = 20
- [ ] 顺子 [2, 3, 4, 5, 6]：基础分 = 20，倍率 = 30，得分 = 600
- [ ] 四条 [K, K, K, K]：基础分 = 40，倍率 = 60，得分 = 2400

---

### 步骤 3.2：实现盲注倍率系统

**目标**：引入盲注概念，增加策略深度

**指令**：
1. 定义盲注等级枚举（小盲注、大盲注、Boss 盲注）
2. 盲注倍率影响：
   - 小盲注：目标分数 ×1，奖励 ×1
   - 大盲注：目标分数 ×2，奖励 ×2
   - Boss 盲注：目标分数 ×3，奖励 ×3 + 特殊规则
3. Boss 盲注特殊规则示例：
   - 某些花色不计分（如：方块不计分）
   - 某些牌型不计分（如：顺子不计分）
   - 出牌数量限制（如：每回合最多出牌 3 次）
4. 修改得分计算：
   ```
   最终得分 = 牌型得分 × 盲注倍率
   ```

**验证测试**：
- [ ] 小盲注关卡：目标分数为基础值
- [ ] 大盲注关卡：目标分数翻倍
- [ ] Boss 盲注：确认特殊规则生效
- [ ] 奖励按倍率正确发放

---

### 步骤 3.3：实现目标分数与过关判定

**目标**：定义胜利条件

**指令**：
1. 为每个关卡定义目标分数
2. 实现回合计数器（当前回合 / 最大回合）
3. 实现累计分数追踪
4. 过关判定：
   - 回合结束前累计分数 ≥ 目标分数 → 胜利
   - 回合用尽且未达标 → 失败
5. 失败后可重新开始本关卡

**验证测试**：
- [ ] 目标 300 分，累计 350 分 → 过关
- [ ] 目标 300 分，最大 3 回合，每回合得分 80，累计 240 → 失败
- [ ] 重新开始后，分数和回合数重置
